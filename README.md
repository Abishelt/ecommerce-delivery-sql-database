# E-commerce & Delivery SQL Database

## Overview
A relational SQLite database designed for a multi-vendor e-commerce and delivery platform. It models customers, orders, payments, shipments, loyalty, and inventory. The project demonstrates skills in database design, normalization, and business reporting queries.

## Tools & Technologies
- SQL (schema design, queries, constraints, indexes)
- SQLite (database engine)
- GitHub + VS Code (version control, reproducible workflows)
- Business Intelligence concepts (reporting queries, KPIs)

## Key Features
- Normalized schema with 22 tables
- Constraints and indexes for data integrity
- Sample inserts for realistic demo data
- Business queries for analytics:
  - Revenue and order trends
  - Top vendors and products
  - Customer loyalty activity
  - Delivery and fulfillment tracking

## Business Value
This database supports smarter decision-making for e-commerce platforms by enabling reporting on sales, customer behavior, and logistics performance. It demonstrates how structured data models can streamline operations and provide actionable insights.

## How to Run
1. Create a new SQLite database
2. Run `/sql/database_schema.sql` to build tables
3. Run `/sql/sample_data.sql` to insert demo data
4. Run `/sql/business_query.sql` to generate analytics

## Learning Outcomes
- Strengthened skills in relational database design and normalization
- Gained experience writing business-focused SQL queries
- Improved ability to document and structure projects for reproducibility
